# 🧩 1. Systemarchitektur

Definiert die grundlegende Struktur des Systems.

**Typische Inhalte:**

* Architekturstile (z. B. Schichtenarchitektur, Microservices)
* Aufteilung in Subsysteme / Module
* Kommunikationsmechanismen (REST, Messaging, etc.)
* Deployment-Struktur (Cloud, On-Premise)

---

# 🏗️ 2. Desgin Prinzipien

Beschreibt Vorgehensweisen bei der Erstellung von Komponenten, Module und Klassen

**Beispiele:**

* Domain Driven Design (DDD) - meist für Backend verwendet
* Component Driven Design (CDD) - meist für Frontend (UI) verwendet

---

# 🔄 3. Datenmodellierung

Definiert, **wie Daten strukturiert und gespeichert werden**.

**Beinhaltet:**

* ER-Diagramme
* Datenbankschema (Tabellen, Relationen)
* Datenflüsse zwischen Komponenten
* Persistenzstrategien (SQL, NoSQL)

---

# 🔌 4. Schnittstellen-Design

Legt fest, **wie externe und interne Systeme kommunizieren**.

**Dazu gehören:**

* REST-APIs / GraphQL
* Datenformate (JSON, XML)
* Authentifizierung & Autorisierung
* API-Dokumentation (OpenAPI/Swagger)

---

# ⚙️ 5. Technologiewahl

Entscheidung für konkrete Technologien und Werkzeuge.

**Beispiele:**

* Programmiersprachen (Java, Dart, C#)
* Frameworks (Spring, Flutter, Unity)
* Datenbanken (PostgreSQL, MongoDB)
* Tools (CI/CD, Docker)

---

# 🔒 6. Qualitätsanforderungen (Non-Functional Requirements)

Beschreibt **wie gut das System sein muss**.

**Typische Aspekte:**

* Performance
* Skalierbarkeit
* Sicherheit
* Wartbarkeit
* Verfügbarkeit

---

# 🔍 7. Systemverhalten (Dynamik)

Wie reagiert das System zur Laufzeit?

**Modelle:**

* Sequenzdiagramme
* Aktivitätsdiagramme
* Zustandsdiagramme
